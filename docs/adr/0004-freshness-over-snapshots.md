# Freshness over snapshots for structured sources

Tool calls to Kartverket, SSB, and MET fetch live, never from snapshots. When a source advertises freshness headers (MET sends `Expires` + `Last-Modified`), the adapter respects them via conditional GET — serve from cache inside `Expires`, revalidate with `If-Modified-Since` after. When it doesn't, the adapter fetches every time; only adopt a TTL if a real rate-limit constraint appears, and document the exception. The cost of stale structured data — wrong kommunenr after a merge, outdated weather — is higher than the cost of an extra HTTP call. Only unstructured text (Wikipedia/arXiv chunks) is indexed or cached aggressively.
