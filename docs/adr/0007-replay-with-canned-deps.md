# Replay drives `handleQuery` with canned deps, not snapshot-diff

Offline replay (`pnpm --filter api replay <report.json>`) re-runs the production `handleQuery` against a recorded eval case, with deps stubbed to return the recorded tool outputs and the recorded **routing plan**. We rejected the obvious alternative of snapshotting the answer and diffing it, because that approach freezes the orchestrator output as the contract — any deliberate change to answer composition would have to be paired with a snapshot-refresh dance. Reusing `handleQuery` means composition changes get re-verified against historical inputs for free. The cost: every probabilistic step in the orchestrator must be expressed as a dep call (so the **router**'s plan is recorded on `QueryResponse.plan`); any future hidden LLM call would silently break replay until extracted.
